# コンパイルエラー修正

## 現状の問題点
1. `ConfigError`の未インポート
2. `Value`型の未インポート
3. `Result`型の使用方法の誤り
4. プライベートメソッド`resolve_alias_path`の使用
5. 型変換の問題（`PathBuf`、`Site`）

## 修正方針
1. 必要なインポートの追加
   - `use crate::config::ConfigError;`
   - `use serde_yaml::Value;`
2. `Result`型の使用を修正
   - カスタム`Result`型の定義を確認
   - エラーハンドリングの修正
3. プライベートメソッドの代替実装
4. 型変換の修正
   - `PathBuf`の変換を適切に行う
   - `Site`型の処理を修正

## 作業難易度
中程度 - 主に型システムとエラーハンドリングの修正が必要

## 修正手順
1. 各ファイルの依存関係を確認
2. 必要なインポートを追加
3. エラーハンドリングのロジックを修正
4. 型変換の問題を解決
