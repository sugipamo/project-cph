# src/contestのコマンド受け入れ実装計画

## 概要
src/contestディレクトリにおいて、commands.yamlで定義されたコマンドを受け入れ、適切に処理する機能を実装します。

## 実装内容

1. コマンドパーサーの実装 (src/contest/parse.rs)
   - commands.yamlの内容を読み込むための構造体の定義
   - コマンド文字列をパースする機能の実装
   - エイリアスの解決機能の実装

2. コマンドモデルの実装 (src/contest/model/mod.rs)
   - コマンド実行に必要なデータ構造の定義
   - コマンドの種類を表す列挙型の定義
   - コマンドのパラメータを表す構造体の定義

3. コマンド実行サービスの実装 (src/contest/service/mod.rs)
   - 各コマンドの実行ロジックの実装
   - エラーハンドリングの実装
   - コマンド実行結果の型定義

4. エラー処理の実装 (src/contest/error.rs)
   - コマンドパース時のエラー定義
   - コマンド実行時のエラー定義
   - エラーメッセージの定義

## テスト計画
1. コマンドパーサーのユニットテスト
2. コマンド実行のユニットテスト
3. エラーケースのテスト
4. 統合テスト

## 実装順序
1. モデルの定義
2. パーサーの実装
3. サービスの実装
4. エラー処理の実装
5. テストの実装
